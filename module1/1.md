# Решение Модуля 1: Базовая настройка сети и сервисов

## 1. Базовая настройка устройств
**Задание:** Настроить имена устройств (FQDN), IPv4 из приватного диапазона (RFC1918) и рассчитать подсети.

### Решение (на примере настройки интерфейсов в Alt Linux):
Настройка выполняется через создание файлов в `/etc/net/ifaces/`.

**Пример для ISP (ens37):**
```bash
# Создание директории интерфейса
mkdir /etc/net/ifaces/ens37

# Настройка параметров (options)
# /etc/net/ifaces/ens37/options
BOOTPROTO=static
TYPE=eth
DISABLED=no
CONFIG_IPv4=yes

# Настройка IP-адреса
# /etc/net/ifaces/ens37/ipv4address
172.16.4.2/28

# Настройка шлюза (при необходимости)
# /etc/net/ifaces/ens37/ipv4route
default via 172.16.4.1

# Перезапуск сети
systemctl restart network
Используйте код с осторожностью.
2. Настройка маршрутизации и NAT на ISP
Задание: Включить Forwarding и настроить динамическую трансляцию (MASQUERADE) для доступа HQ и BR в интернет.
Решение:
Включение пересылки пакетов:
В файле /etc/net/sysctl.conf:
conf
net.ipv4.ip_forward = 1
Используйте код с осторожностью.
Применить: sysctl -p /etc/net/sysctl.conf
Настройка IPTables:
bash
iptables -t nat -A POSTROUTING -o ens33 -s 172.16.4.0/28 -j MASQUERADE
iptables -t nat -A POSTROUTING -o ens33 -s 172.16.5.0/28 -j MASQUERADE
# Сохранение правил
iptables-save > /etc/sysconfig/iptables
systemctl enable --now iptables.service
Используйте код с осторожностью.
3. Учетные записи и безопасность
Задание: Создать пользователей cyberpanda (на серверах) и routerfox (на роутерах) с правами sudo.
Решение:
bash
# Создание пользователя и пароля
useradd -u 1001 cyberpanda
passwd cyberpanda # Пароль: P@ssw0rd

# Добавление в группу wheel для sudo
usermod -aG wheel cyberpanda

# Настройка sudo без пароля (/etc/sudoers)
cyberpanda ALL=(ALL) NOPASSWD:ALL
Используйте код с осторожностью.
4. Защищенный SSH (HQ-SRV и BR-SRV)
Задание: Порт 2020, только для cyberpanda, баннер "Authorized access only".
Решение:
В файле /etc/openssh/sshd_config:
conf
Port 2020
MaxAuthTries 2
AllowUsers cyberpanda
Banner /etc/openssh/bannermotd
Используйте код с осторожностью.
Создание баннера:
bash
echo "Authorized access only" > /etc/openssh/bannermotd
systemctl restart sshd
Используйте код с осторожностью.
5. Настройка IP-туннеля (GRE) между HQ и BR
Задание: Создать туннель между роутерами.
Решение (на HQ-RTR):
bash
mkdir /etc/net/ifaces/gre1
# /etc/net/ifaces/gre1/options
TYPE=iptun
TUNTYPE=gre
TUNLOCAL=172.16.4.2
TUNREMOTE=172.16.5.2
TUNTTL=64
TUNMTU=1476

# /etc/net/ifaces/gre1/ipv4address
10.0.1.1/30 peer 10.0.1.2
Используйте код с осторожностью.
6. Динамическая маршрутизация (OSPF)
Задание: Настроить протокол Link State (FRR/OSPF) через туннель с парольной защитой.
Решение (HQ-RTR):
bash
# Включение OSPF в /etc/frr/daemons
ospfd=yes

# Конфигурация через vtysh
router ospf
 router-id 1.1.1.1
 network 192.168.5.0/29 area 0
 network 192.168.6.0/29 area 0
 network 10.0.1.0/30 area 0

interface gre1
 ip ospf authentication message-digest
 ip ospf message-digest-key 1 md5 P@ssw0rd
Используйте код с осторожностью.
